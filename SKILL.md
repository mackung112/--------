# 🛠️ AI Skill & Technical Standards (Master Prompt)

**เป้าหมาย:** ไฟล์นี้รวบรวม "กฎทางเทคนิค (Technical Rules)" และ "มาตรฐานโค้ด (Coding Standards)" ห้ามละเมิดเด็ดขาด

## 🚨 1. กฎเหล็กทางเทคนิค (Strictly Prohibited)

- **ห้ามใช้ Vanilla JS**: ต้องใช้ React State/Ref/Hooks เท่านั้น (เว้นแต่เป็นส่วนรับ/รันโค้ด อนุญาตให้ใช้ Plugin หรือแพ็กเกจที่คนอื่นสร้างไว้มาใช้แทนได้)
- **ห้ามใช้ Tailwind แบบ Dynamic**: ให้ Hardcode string เต็มรูปแบบ (เช่น `text-indigo-600` ห้ามใช้ `${color}-600`) เพื่อรองรับ Tailwind v4
- **ห้ามแตะ Core Architecture**: ห้ามแก้ไข `\Teach\LMS-React\src\App.jsx`, `\Teach\LMS-React\src\components\LessonViewer.jsx`, `\Teach\LMS-React\vite.config.js` เว้นแต่จะมีระบุในคำสั่ง
- **ห้าม Card-in-Card**: ทุกหน้าต้องเป็นแบบไร้ขอบ (Seamless) ผสานไปกับฉากหลังตาม Immersive Full-Page Standard
- **State Management**: ห้ามใช้ Global State สำหรับ Simulator เด็ดขาด ให้ใช้ Local State (`useState`, `useEffect`) เสมอเพื่อความเป็นอิสระ

## 💻 2. มาตรฐานการเขียนโค้ด (Coding Standards)

- **Component Format**:
  - ให้เขียนเป็น `export default function ComponentName() { ... }`
  - การตั้งชื่อไฟล์: อ้างอิงตามวิชาและหมายเลขบทเรียนลำดับที่ 1 จากหลักสูตร (Curriculum) เสมอ โดยการนำคำย่อของรายวิชา (ตัวพิมพ์เล็ก) มาเชื่อมกับหมายเลขหลักสูตรด้วยเครื่องหมายขีดล่าง `_` และไม่มีคำอธิบายเพิ่มเติม ตัวอย่างเช่น:
    - **วิชา Python (21902-2001)**: หัวข้อ 1.1 ตั้งเป็น `py1_1.jsx`, หัวข้อ 3.5 ตั้งเป็น `py3_5.jsx`
    - **วิชา SQL (21901-2001)**: หัวข้อ 1.2 ตั้งเป็น `sql1_2.jsx`, หัวข้อ 4.8 ตั้งเป็น `sql4_8.jsx`
    - **วิชา OOP (Python OOP)**: หัวข้อ 2.3 ตั้งเป็น `oop2_3.jsx`
    - **วิชา OOAD**: หัวข้อ 3.1 ตั้งเป็น `ooad3_1.jsx`
    - **วิชา DSA**: หัวข้อ 1.1 ตั้งเป็น `dsa1_1.jsx`
- **Interactive Component**: ทุกสื่อ Simulator ให้สร้างเก็บไว้ที่ `\Teach\LMS-React\src\components\interactive\<วิชา>\` (ระบบจะโหลดและนำไปใช้งานอัตโนมัติผ่าน Auto-Registry)
- **ภาษาและเนื้อหา**: โค้ดตัวอย่างในวิชาต้องใช้ **Python** หรือ **MySQL** และคำอธิบายเนื้อหาต้องเป็น **ภาษาไทย** เสมอ
- **JSX Gotchas**: หากมีสัญลักษณ์ `>>>` ในโค้ด ให้ห่อด้วย `{">>>"}` เสมอ
- **เลย์เอาต์แยกการ์ดรายหัวข้อ (Separated Cards Layout)**: ในแต่ละบทเรียนย่อยที่มีหัวข้อย่อยบังคับ (เช่น 4.1.1, 4.1.2, 4.1.3) **ห้าม** ยุบรวมเนื้อหาลงในแท็บ (Tab Navigation) สับเปลี่ยนหน้า ให้ **แยกเนื้อหาออกเป็นบล็อกการ์ดเดี่ยวอิสระเรียงแนวตั้ง (Vertical Stack)** เพื่อให้นักเรียนสามารถเลื่อนอ่านเนื้อหาทั้งหมดได้อย่างลื่นไหล และมีกิจกรรมโต้ตอบที่ชัดเจนในแต่ละหัวข้อย่อยแยกกันอย่างเป็นระบบ

## 🎮 3. แนวทาง Game-Based Learning & Interaction

- **Visual Interactive**: เน้นการสร้าง "Interactive ที่อธิบายภาพและทำงานได้จริง" ให้เห็นชัดเจน ไม่จำเป็นต้องทำระบบสะสมคะแนนที่ซับซ้อน (หมายเหตุ: ถ้าเนื้อหาใน list ระดับที่ 2 ของโฟลเดอร์ `\Teach\LMS-React\docs\curriculum\` สำหรับวิชาต่างๆ เช่น 1.1.1, 1.1.3, 4.5.6, 1.3.8 ไหนสามารถสร้างได้ ให้พยายามสร้างให้ครอบคลุมทุกหัวข้อยิ่งดี)
- **Flowcharts**: ใช้ SVG บริสุทธิ์ แทน Unicode หรือ CSS ให้วาดสวยงามระดับเดียวกับ draw.io
- **Micro-Interactions**: ปุ่มและการ์ดที่โต้ตอบได้ต้องใส่ `active:scale-98 transition-all` เสมอ
- **Controls**: ปุ่ม Play, Pause, และ Reset (ไอคอน `RotateCcw`) ให้พิจารณาใส่ตามความเหมาะสมของสถานการณ์ ไม่จำเป็นต้องบังคับใส่ทุกครั้ง

## 📚 4. หลักสูตรและการจัดการข้อมูล

- **โครงสร้างเนื้อหา**: อ้างอิงโฟลเดอร์ `\Teach\LMS-React\docs\curriculum\`
  - Unit = บทหลัก
  - ลำดับที่ 1 (เช่น 1.1) = จำนวนหน้าเว็บ (Lessons)
  - ลำดับที่ 2 (เช่น 1.1.1) = หัวข้อบังคับที่ต้องมีในหน้านั้น (**กฎเหล็ก: ห้ามใส่ตัวเลขลำดับย่อยเด็ดขาด!** เช่น ในการสร้างหน้า 4.3 เมื่อมีการแยกการ์ดเนื้อหารายหัวข้อ 4.3.1 และ 4.3.2 **ห้ามพิมพ์ตัวเลข 4.3.1 หรือ 4.3.2 ลงในหน้าเว็บ/คอมโพเนนต์เด็ดขาด** ให้ใส่เฉพาะชื่อหัวข้อที่เป็นข้อความเปล่าๆ เท่านั้น เพื่อความสะอาด สวยงาม และตรงตามสไตล์ของระบบ)
- **โครงสร้างข้อมูลบทเรียน (Data Structure Standard)**: ทุกบทเรียนในฐานข้อมูล **ต้องมี** คีย์ข้อมูลให้ครบทั้ง 6 ฟิลด์ ได้แก่ `id`, `title`, `mainTitle`, `subTitle`, `description`, และ `content` วางไว้ในอ็อบเจกต์ `lesson` เสมอ ห้ามขาดแม้แต่ฟิลด์เดียว เพื่อใช้ส่งไปแสดงผลที่ `StandardHeader` ได้อย่างสมบูรณ์และเป็นมาตรฐานเดียวกัน
- **ขั้นตอนการสร้างหน้าใหม่ (New Page Creation Workflow)**:
  1. **สร้างไฟล์ Component**: สร้างไฟล์ใน `\Teach\LMS-React\src\components\interactive\<วิชา>\` โดยตั้งชื่อตามรูปแบบ `<prefix><X>_<Y>.jsx` เท่านั้น (เช่น `py1_1.jsx`, `sql3_5.jsx` อ้างอิงตามวิชาและหมายเลขบทเรียนลำดับที่ 1 จากหลักสูตร) และ **ห้าม** เติมคำอธิบายอื่นในชื่อไฟล์เด็ดขาด
  2. **โครงสร้าง Component**: 
     - **ห้าม** ใส่ส่วนหัว (`<header>`, `h1`, `h2` ที่บอกชื่อเรื่อง) ลงในไฟล์ Component เด็ดขาด เพราะระบบมี `StandardHeader` จัดการให้อัตโนมัติอยู่แล้ว
     - ให้เน้นเขียนเฉพาะส่วนที่เป็น `<main>` หรือตัว Simulator / มินิเกม / เนื้อหา และจบด้วย `<TeacherTask />` เท่านั้น
  3. **เชื่อมโยงข้อมูล**: นำชื่อไฟล์ Component ไปใส่ในรูปแบบ `[Marker]` ในฟิลด์ `content` ของไฟล์ Database (เช่น `pyCourse.js` ในรูปแบบ `[py1_1]` หรือ `[py3_5]`) พร้อมตรวจสอบให้แน่ใจว่าฟิลด์อื่นๆ ทั้ง 5 ฟิลด์ระบุข้อมูลครบถ้วน
  4. **การเขียนทับทันที (Overwrite Policy - ห้ามเปิดอ่านไฟล์เก่าเด็ดขาด)**: เมื่อผู้ใช้ระบุหรือสั่งให้สร้างหน้าบทเรียนใดขึ้นมาใหม่ ให้ AI ดำเนินการคิดออกแบบและสร้างโครงสร้างคอมโพเนนต์ใหม่ 100% ขึ้นมาเองตั้งแต่ต้น แล้วเขียนทับ (Overwrite) ลงบนไฟล์เดิมได้เลยทันที โดย**ห้ามเรียกใช้เครื่องมือเปิดอ่านไฟล์เก่าตัวเดิมก่อนลงมือทำอย่างเด็ดขาด** (ไม่จำเป็นและห้ามพยายามรักษาโครงสร้างหรือโค้ดเดิมใดๆ ไว้ เพื่อประสิทธิภาพและป้องกันไม่ให้ดีไซน์เดิมมารบกวนรูปแบบใหม่ที่สวยงามกว่าเดิม)

- **ปรับตัวตามผู้ใช้**: สังเกตและจดจำรูปแบบการเขียนโค้ดหรือตรรกะที่ผู้ใช้พึงพอใจ หากผู้ใช้สั่งให้หลีกเลี่ยงการเขียนแบบใด หรือให้แก้การทำงานซ้ำๆ ให้คุณถือเป็น "บรรทัดฐานใหม่ (New Standard)" และเขียนทับลงในหมวดหมู่นี้ได้ทันที เพื่อผลิตผลงานให้ตรงใจมากที่สุด

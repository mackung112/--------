# 🤖 ศูนย์กลางปฏิบัติการ AI (AI OS & Agent Hub)

ยินดีต้อนรับสู่โปรเจกต์ **"ห้องเรียนครูแม็ค"** (LMS Platform)
ไฟล์นี้คือ "จุดเริ่มต้น (Entry Point)" ที่ออกแบบมาให้อ่านง่าย กระชับ และเน้นการเรียนรู้พฤติกรรม (Training) เพื่อให้ AI เข้าใจและสร้างผลลัพธ์ได้ตรงใจผู้ใช้มากที่สุด

---

## 📚 1. ฐานข้อมูลที่ต้องอ่าน (Knowledge Base)
เมื่อเริ่มต้นทำงาน คุณไม่จำเป็นต้องจำทุกอย่าง แต่ **ต้อง** อ้างอิงไฟล์เหล่านี้เสมอ:
- 📌 **[\Teach\SKILL.md](\Teach\SKILL.md)**: กฎเหล็ก และข้อห้ามเด็ดขาด (Master Prompt)
- 📌 **[\Teach\DESIGN.md](\Teach\DESIGN.md)**: สเปก UI/UX และระบบโครงสร้างหน้า (Immersive Full-Page)
- 📌 **[\Teach\CONTEXT.md](\Teach\CONTEXT.md)**: โครงสร้างไฟล์และประวัติโปรเจกต์ (Long-term Memory)
- 📌 **[\Teach\task.md](\Teach\task.md)**: แผนงานและเช็คลิสต์ปัจจุบัน (Short-term Memory)

## 🧠 2. ระบบเรียนรู้และปรับตัวอย่างต่อเนื่อง (Continuous Training Data)
*(AI ต้องสังเกตความชอบของผู้ใช้ และสามารถอัปเดตข้อมูลในส่วนนี้ได้เสมอเมื่อเรียนรู้สิ่งใหม่)*

เพื่อให้ผลลัพธ์ **"ตรงใจผู้ใช้มากที่สุด"** ให้ยึดพฤติกรรมหลักดังนี้:
1. **คุณภาพระดับพรีเมียม**: ห้ามทำระบบพื้นฐาน (Basic Card/List) ทุกงานต้องมี Gamification หรือ Simulator ขั้นสูงที่ดูแพงและมีลูกเล่นเสมอ
2. **ห้ามทิ้งงานกลางทาง (No Placeholders)**: ห้ามเขียนโค้ดแบบย่อแล้วบอกให้ผู้ใช้ไปเติมเอง ต้องเขียนโค้ดให้เสร็จสมบูรณ์ 100% รันได้จริง
3. **ใส่ใจ Interaction**: ทุกปุ่ม ทุกการ์ด ต้องมีเอฟเฟกต์การกด (`active:scale-98`) และมีแสงเงาที่สอดคล้องกับธีม
4. **ไม่ถามในสิ่งที่ระบบมีอยู่แล้ว**: ให้อ่านไฟล์ Database (`\Teach\LMS-React\src\data.js`, `\Teach\LMS-React\docs\curriculum\`) ให้ละเอียดก่อนลงมือทำ
5. **ปรับตัวจาก Feedback**: หากผู้ใช้สั่งแก้สิ่งใดบ่อยๆ ให้จดจำและนำมาประยุกต์ใช้กับคอมโพเนนต์อื่นๆ ทันที
6. **อัปเดตความจำถาวร (Self-Updating)**: หากคุณเรียนรู้รสนิยมใหม่ สไตล์โค้ด หรือโครงสร้างใหม่จากผู้ใช้ คุณ **ต้อง** นำสิ่งนั้นไปเขียนอัปเดตลงในไฟล์ตั้งค่าที่เกี่ยวข้องด้วยตัวเองทันที (เช่น แก้ไข `\Teach\SKILL.md` สำหรับเทคนิคโค้ด, `\Teach\DESIGN.md` สำหรับสี/UI, หรือ `\Teach\CONTEXT.md` สำหรับโครงสร้าง/ประวัติ) เพื่อให้ข้อมูลที่ผู้ใช้ชอบล่าสุดถูกบันทึกไว้อย่างถาวร

## 🚀 3. กระบวนการทำงาน (Workflow Handoff)
- **เริ่มงาน**: เช็ค `\Teach\task.md` แล้วลงมือทำตาม Phase ทันที
- **ระหว่างงาน**: 
  - ใช้ **Subagents** (`invoke_subagent`) หากต้องรีเสิร์ชโค้ดจำนวนมาก
  - เขียนแผนงานลง `\Teach\implementation_plan.md` (Planning Mode) หากต้องรื้อโครงสร้างใหญ่
- **จบงาน (สำคัญมาก)**: อัปเดต `[x]` ใน `\Teach\task.md` และเขียนสรุปส่งมอบงานไว้ที่ด้านล่างสุดของ `\Teach\task.md` ทุกครั้งก่อนจบเซสชัน
- **☁️ การดึงข้อมูลข้ามเครื่อง**: ข้อมูลล่าสุดของโปรเจกต์จะถูกเก็บไว้ที่ `https://github.com/mackung112/Teach.git` เสมอ หากผู้ใช้สั่งว่า **"ดึงข้อมูลล่าสุด"** ให้รันคำสั่ง Git (เช่น `git fetch origin` และ `git reset --hard origin/main`) เพื่อดึงข้อมูลมาทับในเครื่องทันที

# ระบบปฏิบัติการ บำรุงรักษา และเครือข่ายคอมพิวเตอร์เบื้องต้น (Integrated IT: 31910-0001 & 31910-0005)

## Unit 1: พื้นฐานระบบคอมพิวเตอร์และเครือข่าย

- 1.1 องค์ประกอบของระบบคอมพิวเตอร์เชิงลึก
  - 1.1.1 ความหมาย หน้าที่ และบทบาทของระบบคอมพิวเตอร์ในยุคดิจิทัลธุรกิจ
  - 1.1.2 ฮาร์ดแวร์ (Hardware) และเจาะลึกหน่วยการทำงานทั้ง 4 (Input, Process, Storage, Output)
  - 1.1.3 ซอฟต์แวร์ (Software) ประเภทต่างๆ (System Software, Application Software, Utility Software)
  - 1.1.4 บุคลากรคอมพิวเตอร์ (Peopleware) ระดับต่างๆ และบทบาทหน้าที่ในองค์กร
  - 1.1.5 ข้อมูลและสารสนเทศ (Data & Information) และวงจรการประมวลผลข้อมูล (Data Processing Cycle)
  - 1.1.6 ความแตกต่างระหว่างข้อมูลดิบ (Raw Data) และสารสนเทศเชิงประกอบการตัดสินใจธุรกิจ
  - 1.1.7 ประโยชน์และบทบาทของระบบคอมพิวเตอร์ในอุตสาหกรรมอัจฉริยะยุค 4.0
- 1.2 ความรู้เบื้องต้นเกี่ยวกับระบบเครือข่ายคอมพิวเตอร์
  - 1.2.1 ความหมายและวัตถุประสงค์หลักของการสร้างระบบเครือข่ายคอมพิวเตอร์
  - 1.2.2 องค์ประกอบพื้นฐานของการสื่อสารข้อมูล (Sender, Receiver, Message, Medium, Protocol)
  - 1.2.3 ประโยชน์ของการแชร์ทรัพยากรแบบเจาะจง (Hardware sharing, Software sharing, Data sharing)
  - 1.2.4 ข้อจำกัด ความเสี่ยง และต้นทุนในการวางระบบเครือข่ายสำหรับองค์กร
  - 1.2.5 ทิศทางการส่งสัญญาณข้อมูล (Transmission Mode: Simplex, Half-Duplex, Full-Duplex)
- 1.3 ประเภทและโครงสร้างของระบบเครือข่าย
  - 1.3.1 การจำแนกเครือข่ายตามระยะทางและขนาดพื้นที่ (PAN, LAN, MAN, WAN) พร้อมตัวอย่างการใช้งานจริง
  - 1.3.2 โครงสร้างเครือข่ายแบบพึ่งพาเครื่องแม่ข่าย (Client-Server) และการเปรียบเทียบข้อดี-ข้อเสีย
  - 1.3.3 โครงสร้างเครือข่ายแบบเท่าเทียมกัน (Peer-to-Peer) และการเปรียบเทียบข้อดี-ข้อเสีย
  - 1.3.4 ขอบข่ายของสถาปัตยกรรมเครือข่ายการแชร์ข้อมูลภายในและภายนอก (Intranet, Extranet, Internet)
- 1.4 รูปแบบการเชื่อมต่อเครือข่าย (Network Topology)
  - 1.4.1 การเชื่อมต่อแบบบัส (Bus Topology) ลักษณะทางกายภาพและข้อจำกัดการชนกันของข้อมูล
  - 1.4.2 การเชื่อมต่อแบบดาว (Star Topology) การใช้อุปกรณ์ศูนย์กลางและจุดล้มเหลวเดี่ยว (Single Point of Failure)
  - 1.4.3 การเชื่อมต่อแบบวงแหวน (Ring Topology) และระบบการส่งโทเคน (Token passing) เพื่อสิทธิ์การคุย
  - 1.4.4 การเชื่อมต่อแบบเมช (Mesh Topology) แบบเต็มรูปแบบ (Full Mesh) และบางส่วน (Partial Mesh)
  - 1.4.5 การเชื่อมต่อแบบผสม (Hybrid Topology) สำหรับองค์กรขนาดใหญ่และการออกแบบที่มีความยืดหยุ่นสูง
  - 1.4.6 การวิเคราะห์เปรียบเทียบต้นทุน ประสิทธิภาพ และความเหมาะสมในการเลือกติดตั้งแต่ละ Topology

ใบงานปฏิบัติ 1: สำรวจฮาร์ดแวร์คอมพิวเตอร์ในห้องปฏิบัติการ ค้นหาว่าอุปกรณ์นั้นคืออะไร มีหน้าที่อะไร และวิธีใช้งาน มาคนละ 10 ชิ้น

## Unit 2: ฮาร์ดแวร์และการประกอบคอมพิวเตอร์

- 2.1 เจาะลึกอุปกรณ์และชิ้นส่วนคอมพิวเตอร์ (Hardware Components)
  - 2.1.1 หน่วยประมวลผลกลาง (CPU) สถาปัตยกรรม (Cores/Threads), ซ็อกเก็ต (Socket), แคช (Cache) และระบบระบายความร้อน
  - 2.1.2 แผงวงจรหลัก (Motherboard) ฟอร์มแฟกเตอร์ (ATX, Micro-ATX), ชิปเซ็ต, สล็อตขยาย (PCIe) และพอร์ต I/O
  - 2.1.3 หน่วยความจำหลัก (RAM) ประเภทเทคโนโลยี (DDR4, DDR5), บัส (Bus speed), และค่าความจุของหน่วยความจำ
  - 2.1.4 อุปกรณ์จัดเก็บข้อมูล (Storage) ความแตกต่างเชิงโครงสร้างและความเร็วระหว่าง HDD, SSD (SATA, M.2, NVMe)
  - 2.1.5 แหล่งจ่ายไฟ (Power Supply Unit - PSU) ขนาดวัตต์, มาตรฐาน 80 Plus, ถอดสายได้/ไม่ได้ และการคำนวณการใช้ไฟของระบบ
  - 2.1.6 การ์ดแสดงผล (VGA/GPU) และการเลือกใช้อุปกรณ์ต่อพ่วงภายนอกให้เหมาะสมกับงานแต่ละประเภท
  - 2.1.7 เคสคอมพิวเตอร์ (Computer Case/Chassis) ฟอร์มแฟกเตอร์ การระบายอากาศภายใน และการติดตั้งแผงเชื่อมต่อแผงหน้า
  - 2.1.8 ชนิดและประเภทของจอแสดงผล (Monitor Panel: IPS, VA, TN) และสายเชื่อมต่อสัญญาณภาพ (HDMI, DisplayPort, VGA)
- 2.2 เครื่องมือและข้อควรระวังในการประกอบและซ่อมบำรุง
  - 2.2.1 เครื่องมือช่างพื้นฐานสำหรับงานคอมพิวเตอร์ (ไขควง, คีม, ซิลิโคนระบายความร้อน, สายรัด)
  - 2.2.2 ความเข้าใจเรื่องอันตรายจากไฟฟ้าสถิต (Electrostatic Discharge - ESD) ที่มีผลต่อวงจรอิเล็กทรอนิกส์
  - 2.2.3 วิธีการป้องกัน ESD (การใช้ Anti-static wrist strap, เสื่อกันไฟฟ้าสถิต และข้อควรปฏิบัติตัว)
  - 2.2.4 ข้อควรระวังด้านความปลอดภัยต่อชีวิตและอุปกรณ์ในการทำงานเกี่ยวกับแรงดันไฟฟ้ากระแสสลับ
- 2.3 ขั้นตอนการประกอบคอมพิวเตอร์อย่างเป็นระบบ (Step-by-Step Assembly)
  - 2.3.1 การเตรียมพื้นที่ทำงานที่สะอาดและปลอดภัย การแกะกล่องชิ้นส่วน และการเตรียมเคส (Case preparation)
  - 2.3.2 การติดตั้ง CPU ลงบนซ็อกเก็ต, การทาซิลิโคน (Thermal Paste) และการติดตั้งชุดระบายความร้อน (CPU Cooler)
  - 2.3.3 การติดตั้ง RAM ลงบนสล็อตตามคู่มือเมนบอร์ดเพื่อรองรับระบบ Dual-Channel
  - 2.3.4 การติดตั้งอุปกรณ์จัดเก็บข้อมูล (M.2 SSD บนบอร์ด หรือ HDD/SSD ในช่องไดรฟ์ของเคส)
  - 2.3.5 การวางเมนบอร์ด (Motherboard) ลงในเคสอย่างถูกต้องและการยึดเกลียวน็อตรอง (Standoffs)
  - 2.3.6 การติดตั้ง Power Supply และการเดินสายไฟเลี้ยงหลัก (24-pin ATX, 8-pin EPS, PCIe power)
  - 2.3.7 การเชื่อมต่อสายสัญญาณแผงด้านหน้า (Front Panel Header: Power SW, Reset SW, HDD LED), USB, Audio และพัดลมเคส
  - 2.3.8 ศิลปะการจัดสายสัญญาณ (Cable Management) เพื่อความสวยงามและความเป็นระเบียบของทิศทางลม (Airflow)
  - 2.3.9 การติดตั้งการ์ดแสดงผล (GPU) และอุปกรณ์ขยายสัญญาณภาพลงบนสล็อต PCIe พร้อมล็อกสลักและต่อไฟเลี้ยง
- 2.4 การตรวจสอบและทดสอบระบบเบื้องต้น (Initial Testing)
  - 2.4.1 กระบวนการ Power-On Self-Test (POST) การทำงานและวัตถุประสงค์ของการทดสอบระดับต่ำ
  - 2.4.2 การแปลความหมายของเสียงสัญญาณเตือน Beep Code หรือสัญลักษณ์ไฟ Debug LED บนเมนบอร์ดเพื่อหาจุดเสีย
  - 2.4.3 การเข้าสู่หน้าจอ BIOS/UEFI เพื่อตรวจสอบสถานะการมองเห็นฮาร์ดแวร์ (CPU Temp, RAM speed, Boot drives)
  - 2.4.4 การตรวจสอบระบบระบายความร้อน รอบพัดลม และค่าแรงดันไฟในหน้าแรกของ UEFI/BIOS
  - 2.4.5 การตั้งค่าพื้นฐานใน BIOS เช่น ลำดับการบูต (Boot Sequence) และการเปิดใช้งาน XMP/EXPO เพื่อความเร็วแรมเต็มประสิทธิภาพ

ใบงานปฏิบัติ 2: ถอดประกอบเครื่องคอมพิวเตอร์จริงตามขั้นตอนมาตรฐาน การจัดสายไฟ (Cable Management) และทดสอบเปิดเครื่องเพื่อตรวจสอบสถานะฮาร์ดแวร์ในหน้า UEFI/BIOS

## Unit 3: ระบบปฏิบัติการและการติดตั้ง

- 3.1 สถาปัตยกรรมและหลักการทำงานของระบบปฏิบัติการ
  - 3.1.1 ความหมาย หน้าที่ และบทบาทของ OS ในฐานะตัวกลางระดับโครงสร้างระหว่างฮาร์ดแวร์และผู้ใช้
  - 3.1.2 โครงสร้างส่วนประกอบของ OS (Kernel, Shell, User Interface: GUI/CLI)
  - 3.1.3 การบริหารจัดการทรัพยากรหลักของ OS (Process Management, Memory Management, Device Management, File Management)
  - 3.1.4 การเปรียบเทียบตระกูล OS เชิงธุรกิจ (Windows, macOS, Linux distributions) และรุ่น Client เทียบกับ Server
  - 3.1.5 สถาปัตยกรรมระบบ 32-bit (x86) เปรียบเทียบกับ 64-bit (x64) และระบบปฏิบัติการแบบ Multi-Tasking/Multi-User
- 3.2 การเตรียมการก่อนติดตั้งระบบปฏิบัติการ
  - 3.2.1 การอ่านสเปคฮาร์ดแวร์และประเมินความต้องการระบบขั้นต่ำ (System Requirements)
  - 3.2.2 การสร้างสื่อติดตั้ง (Bootable USB) โดยใช้เครื่องมือมาตรฐาน (เช่น Rufus, Windows Media Creation Tool)
  - 3.2.3 ทฤษฎีระบบไฟล์ (File System: FAT32, NTFS, exFAT, ext4) และความสามารถในการรองรับขนาดไฟล์
  - 3.2.4 ตารางพาร์ติชันแบบ MBR (Master Boot Record) เปรียบเทียบเชิงลึกกับ GPT (GUID Partition Table)
  - 3.2.5 การตั้งค่า BIOS/UEFI สำหรับการบูตติดตั้ง (Boot Sequence/Priority, Secure Boot, TPM)
  - 3.2.6 ความเข้าใจและการเตรียมตัวเกี่ยวกับลิขสิทธิ์สิทธิ์การใช้งาน (Operating System Licensing: Retail, OEM, Volume License)
- 3.3 กระบวนการติดตั้งระบบปฏิบัติการและตั้งค่าพื้นฐาน
  - 3.3.1 ขั้นตอนการติดตั้ง OS แบบล้างระบบ Clean Install (ตัวอย่างระบบปฏิบัติการ Windows)
  - 3.3.2 การลบ สร้าง ปรับปรุงขนาด และแบ่งพาร์ติชัน (Partitioning) ระหว่างกระบวนการจัดการดิสก์ในการติดตั้ง
  - 3.3.3 การตั้งค่าระบบเบื้องต้นระยะ Out-of-Box Experience (OOBE) (ภูมิภาค, ภาษา, รูปแบบแป้นพิมพ์, วันเวลา)
  - 3.3.4 การสร้างบัญชีผู้ใช้งานระบบ (Local Account เปรียบเทียบกับ Microsoft/Cloud Account)
  - 3.3.5 การปรับแต่งค่าการแชร์ความเป็นส่วนตัว (Privacy Settings) และการตั้งค่าความปลอดภัยรหัส PIN
- 3.4 การจัดการไดรเวอร์ (Device Drivers) และการอัปเดต
  - 3.4.1 ความหมายและหน้าที่ของ Driver ในฐานะล่ามแปลคำสั่งที่มีต่อฮาร์ดแวร์แต่ละชิ้น
  - 3.4.2 การตรวจสอบอุปกรณ์ที่ชำรุดหรือยังไม่มี Driver ผ่านเครื่องมือ Device Manager
  - 3.4.3 วิธีการค้นหา ดาวน์โหลดตรงจากเว็บไซต์ผู้ผลิต และการติดตั้ง Driver (VGA, Chipset, Audio, Network)
  - 3.4.4 การใช้ระบบ Windows Update เพื่อความสมบูรณ์ของระบบ ความปลอดภัย และการอุดช่องโหว่ (Patches)
  - 3.4.5 การจัดการข้อผิดพลาดของไดรเวอร์เบื้องต้น (Roll Back Driver, Disable/Enable Device และ Uninstall Driver)

ใบงานปฏิบัติ 3: สร้าง Bootable USB, ตั้งค่า BIOS/UEFI ให้บูตจาก USB, ดำเนินการติดตั้ง Windows OS แบบแบ่งพาร์ติชัน และติดตั้ง Driver อุปกรณ์ให้ครบถ้วนไม่มีเครื่องหมายตกใจใน Device Manager

## Unit 4: อุปกรณ์และสื่อกลางในระบบเครือข่าย

- 4.1 อุปกรณ์เครือข่ายระดับฮาร์ดแวร์พื้นฐาน (Network Devices)
  - 4.1.1 การ์ดเครือข่าย (Network Interface Card - NIC) หน้าที่ โครงสร้าง และความหมายของหมายเลขทางกายภาพ MAC Address
  - 4.1.2 หลักการทำงานของฮับ (Hub) และรีพีตเตอร์ (Repeater) การแพร่กระจายสัญญาณข้ามช่องใน Physical Layer
  - 4.1.3 หลักการทำงานของสวิตช์ (Switch) การอ่าน MAC Address Table และการรับส่งข้อมูลส่วนบุคคลใน Data Link Layer
  - 4.1.4 เราเตอร์ (Router) หน้าที่การวิเคราะห์และนำหาเส้นทางข้ามเครือข่ายและการทำงานใน Network Layer
  - 4.1.5 อุปกรณ์กระจายสัญญาณไร้สาย (Wireless Access Point - WAP) และหลักการส่งคลื่นสัญญาณผ่านอากาศ
  - 4.1.6 ไฟร์วอลล์ (Firewall) ระดับฮาร์ดแวร์เบื้องต้น สำหรับควบคุมความปลอดภัยขอบเขตเครือข่าย
  - 4.1.7 โมเด็ม (Modem) หน้าที่การแปลงสัญญาณอนาล็อกและดิจิทัลในการเชื่อมต่อสายผู้ให้บริการภายนอก
- 4.2 สื่อกลางการเชื่อมต่อแบบใช้สาย (Wired Transmission Media)
  - 4.2.1 สายสัญญาณคู่ตีเกลียว (Twisted Pair: UTP และ STP) คุณลักษณะ โครงสร้างภายใน และการป้องกันสัญญาณรบกวน
  - 4.2.2 หมวดหมู่ของสาย UTP (Category 5e, 6, 6a) ขอบเขตความเร็ว แบนด์วิดท์ และระยะทางเชื่อมโยงสูงสุด
  - 4.2.3 โครงสร้างและคุณสมบัติทางเคมี/ฟิสิกส์ของสายโคแอกเชียล (Coaxial Cable) ในการส่งสัญญาณวิทยุและเคเบิล
  - 4.2.4 สายใยแก้วนำแสง (Fiber Optic Cable) โครงสร้างแกนแก้ว, โหมดการส่ง (Single-mode vs Multi-mode) และการนำไปใช้
  - 4.2.5 ปัญหาการเสื่อมถอยของสัญญาณตามระยะทาง (Attenuation) และสัญญาณรบกวนข้ามคู่สาย (Crosstalk) ในสายทองแดง
- 4.3 การเข้าหัวสายสัญญาณ UTP และการทดสอบ
  - 4.3.1 โครงสร้างของหัวต่อ RJ-45 และแจ็ครับสัญญาณติดผนังตัวเมีย
  - 4.3.2 มาตรฐานสากลการเรียงสีสายสัญญาณคู่ตีเกลียว (TIA/EIA-568A และ TIA/EIA-568B)
  - 4.3.3 ขั้นตอนการปอกสายไฟและเข้าหัวแบบสายตรง (Straight-through Cable) สำหรับต่อพ่วงระหว่างอุปกรณ์ต่างชนิดกัน
  - 4.3.4 ขั้นตอนการปอกสายไฟและเข้าหัวแบบสายไขว้ (Crossover Cable) สำหรับต่อเชื่อมระหว่างอุปกรณ์ชนิดเดียวกันโดยตรง
  - 4.3.5 การใช้เครื่องมือทดสอบสายสัญญาณ (Cable Tester) เพื่อวิเคราะห์ลำดับสีความถูกต้องและจุดสายสัญญาณขาด
  - 4.3.6 การเข้าสาย UTP กับเต้ารับตัวเมีย (RJ-45 Keystone Jack / Wall Outlet) และแผงกระจายสายสัญญาณ (Patch Panel)
- 4.4 สื่อกลางการเชื่อมต่อแบบไร้สาย (Wireless Transmission Media)
  - 4.4.1 คุณสมบัติทางฟิสิกส์ของคลื่นวิทยุ คลื่นไมโครเวฟ และอินฟราเรด ในการรับส่งข้อมูลผ่านตัวกลางไร้สาย
  - 4.4.2 มาตรฐานเครือข่ายไร้สาย IEEE 802.11 (ประวัติย้อนหลังและปัจจุบัน: Wi-Fi 4, 5, 6) ความถี่ 2.4GHz และ 5GHz
  - 4.4.3 มาตรฐานและขอบข่ายเทคโนโลยีการเชื่อมต่อ Bluetooth เพื่ออุปกรณ์ส่วนบุคคล
  - 4.4.4 มาตรฐานระบบความปลอดภัยและการเข้ารหัสความปลอดภัยไร้สายเบื้องต้น (WEP, WPA, WPA2, WPA3)
  - 4.4.5 ปัจจัยที่มีผลต่อคุณภาพสัญญาณไร้สาย (สิ่งกีดขวางความหนาแน่น, คลื่นทับซ้อน, ระยะห่าง) และการปรับเสาอากาศ

ใบงานปฏิบัติ 4: ปฏิบัติการปอกสาย LAN เข้าหัว RJ-45 ด้วยคีมย้ำสาย ทั้งรูปแบบสายตรง (Straight-through) และสายไขว้ (Crossover) พร้อมใช้ Cable Tester ทดสอบความสมบูรณ์ของสัญญาณทั้ง 8 เส้น

## Unit 5: โปรโตคอล การกำหนด IP Address และการเชื่อมต่อ

- 5.1 มาตรฐานและแบบจำลองการสื่อสารข้อมูล
  - 5.1.1 ความหมายและหน้าที่ของโปรโตคอล (Protocol) กฎระเบียบกลางเพื่อให้ระบบปฏิบัติการคุยกันรู้เรื่อง
  - 5.1.2 แบบจำลองโครงสร้างอ้างอิง OSI (OSI Reference Model) เจาะลึกการทำงานและหน้าที่ทีละชั้นทั้ง 7 Layers
  - 5.1.3 ชุดโปรโตคอล TCP/IP (TCP/IP Model) 4 Layers และการนำไปวิเคราะห์เทียบเคียงสอดคล้องกับ OSI Model
  - 5.1.4 หน้าที่ ขอบข่ายการทำงาน และพอร์ตมาตรฐานของโปรโตคอลในเครือข่าย (HTTP/HTTPS, FTP, DNS, DHCP, SSH, Telnet)
  - 5.1.5 ความหมายและโครงสร้างของหน่วยข้อมูลในการส่งผ่านข้ามโครงข่าย (Data Encapsulation และ Decapsulation)
- 5.2 โครงสร้างและการจัดสรร Internet Protocol (IPv4 Address)
  - 5.2.1 โครงสร้างเชิงลึกของ IPv4 Address (ขนาด 32 บิต) การจำแนกส่วนที่เป็น Network ID และ Host ID
  - 5.2.2 การจำแนกคลาสมาตรฐานของ IPv4 (Class A, B, C, D, E) และขอบเขตการคำนวณช่วงหมายเลขที่นำมาใช้ได้จริง
  - 5.2.3 ความแตกต่างระหว่างหมายเลขไอพีสาธารณะ (Public IP) และไอพีส่วนตัว (Private IP) รวมทั้ง Loopback Address
  - 5.2.4 ความสำคัญและกลไกการทำงานของหน้ากากเครือข่าย Subnet Mask และเทคนิคการเปรียบเทียบตรรกะระดับบิต ANDing
  - 5.2.5 บทบาทและความสำคัญของเกตเวย์เริ่มต้น Default Gateway ในการนำส่งข้อมูลข้ามวงเครือข่าย
  - 5.2.6 บทแนะนำเทคโนโลยี IPv6 (ขนาด 128 บิต) เหตุผลความจำเป็นในการเปลี่ยนผ่าน และการเปรียบเทียบเบื้องต้นกับ IPv4
- 5.3 การคำนวณและแบ่งเครือข่ายย่อย (Subnetting)
  - 5.3.1 วัตถุประสงค์และประโยชน์ของการแบ่งวงเครือข่ายย่อย (Subnetting) เพื่อความปลอดภัยและการจราจรที่ดีในองค์กร
  - 5.3.2 การคำนวณและแบ่งเครือข่ายย่อยแบบ FLSM (Fixed Length Subnet Mask) สำหรับหน้ากากย่อยคงที่
  - 5.3.3 การคำนวณและแบ่งเครือข่ายย่อยแบบ VLSM (Variable Length Subnet Mask) เพื่อแบ่งกลุ่มตามจำนวนเครื่องอย่างคุ้มค่า
  - 5.3.4 รูปแบบการเขียนคำย่อแบบ CIDR (Classless Inter-Domain Routing) และการแปลงค่าหน้ากากเครือข่ายแบบเลขฐานสอง
- 5.4 การตั้งค่าเครือข่าย การทดสอบ และการแชร์ทรัพยากร
  - 5.4.1 การกำหนดเครือข่ายบน OS: การระบุแบบแมนนวล Static IP และการรับสิทธิ์ค่าอัตโนมัติ Dynamic (DHCP)
  - 5.4.2 การใช้เครื่องมือตรวจสอบหมายเลข IP ปัจจุบันและสถานะฮาร์ดแวร์แลน (Command: ipconfig /all, ifconfig, ip a)
  - 5.4.3 การใช้เครื่องมือตรวจสอบการเชื่อมต่อ (Command: ping, tracert/traceroute) เพื่อทดสอบความเสถียรและทิศทางการส่งข้อมูล
  - 5.4.4 หลักการและการตั้งค่าแชร์ไฟล์ (File Sharing) ในวงแลน รวมถึงความสำคัญของสิทธิ์สิทธิ์การเข้าถึง (NTFS Permissions)
  - 5.4.5 การแชร์เครื่องพิมพ์ร่วมใช้งาน (Printer Sharing) ให้เครื่องปลายทางบนระบบเครือข่าย
  - 5.4.6 การใช้เครื่องมือตรวจสอบการเชื่อมต่อเครือข่ายบนเว็บบราวเซอร์เบื้องต้นและการแก้ปัญหาเบื้องต้นเกี่ยวกับการตั้งค่า DNS

ใบงานปฏิบัติ 5: ออกแบบหมายเลข IP (Subnetting เบื้องต้น) ตั้งค่า Static IP Address ให้กับเครื่องคอมพิวเตอร์ ทดสอบการเชื่อมต่อด้วยคำสั่ง Ping และทำการตั้งค่าแชร์โฟลเดอร์ให้เครื่องอื่นสามารถดึงข้อมูลได้

## Unit 6: การบำรุงรักษาและการใช้โปรแกรมอรรถประโยชน์

- 6.1 การใช้โปรแกรมอรรถประโยชน์ (Utility Programs) สำหรับระบบและไฟล์
  - 6.1.1 ความหมาย หน้าที่ และขอบเขตประเภทของซอฟต์แวร์ระบบอรรถประโยชน์
  - 6.1.2 การใช้งานซอฟต์แวร์บีบอัดจัดการไฟล์ (Zip/Unzip utilities เช่น 7-Zip, WinRAR) เพื่อส่งไฟล์เครือข่ายสะดวก
  - 6.1.3 การจัดการพื้นที่จัดเก็บและทำความสะอาดระบบไฟล์ (Disk Cleanup, Storage Sense) เพื่อล้างไฟล์แคชและขยะระบบ
  - 6.1.4 หลักการจัดระเบียบเศษไฟล์ข้อมูล (Disk Defragmentation) สำหรับจานหมุน HDD และคำเตือนห้ามจัดแฟรกเมนต์บน SSD
  - 6.1.5 การตรวจสอบความสมบูรณ์ของระบบไฟล์ระบบด้วยโปรแกรมอรรถประโยชน์บรรทัดคำสั่ง (เช่น sfc /scannow และ chkdsk)
- 6.2 การรักษาความปลอดภัย การสำรอง และการกู้คืนข้อมูล
  - 6.2.1 รู้จักลักษณะและพฤติกรรมของภัยคุกคามไซเบอร์พื้นฐาน (Virus, Worm, Trojan, Ransomware, Spyware)
  - 6.2.2 การติดตั้ง อัปเดต และสแกนค้นหาสิ่งแปลกปลอมด้วยโปรแกรมแอนตี้ไวรัส (Windows Defender / Antivirus ภายนอก)
  - 6.2.3 ทฤษฎีความสำคัญและแนวทางปฏิบัติการสำรองระบบไฟล์ข้อมูล (Backup: Full, Incremental, Differential) และกฎ 3-2-1
  - 6.2.4 การสร้างและการกู้กลับย้อนคืนด้วย System Restore Point บนสภาพแวดล้อม Windows OS
  - 6.2.5 หลักการกู้คืนข้อมูลไฟล์ที่ถูกลบไปแล้ว (Data Recovery) และขั้นตอนปฏิบัติตามมาตรฐานป้องกันความปลอดภัยข้อมูลสูญหาย
  - 6.2.6 การสร้างบัญชีผู้ใช้งาน สิทธิ์จำกัดการใช้เครื่อง (User Account Management) และมาตรฐานการสร้างรหัสผ่านยากแก่การเดา
  - 6.2.7 จริยธรรมและกฎหมายที่เกี่ยวเนื่องกับพระราชบัญญัติว่าด้วยการกระทำความผิดเกี่ยวกับคอมพิวเตอร์เบื้องต้นที่ช่างต้องระวัง
- 6.3 การบำรุงรักษาฮาร์ดแวร์คอมพิวเตอร์เชิงป้องกัน (Preventive Maintenance)
  - 6.3.1 ความสำคัญ วงรอบขั้นตอนเวลาการวางระบบบำรุงรักษาเชิงป้องกันทางกายภาพสำหรับเครื่องใช้ไฟฟ้าสำนักงาน
  - 6.3.2 เครื่องมือเทคนิคทำความสะอาดพื้นผิวภายนอกคอมพิวเตอร์ แผงจอภาพ และแป้นปุ่มสัมผัสอย่างปลอดภัยถูกขั้นตอน
  - 6.3.3 การถอดเป่าลมและดูดฝุ่นสะสมตามแผงพัดลมระบายความร้อน ตะแกรงเครื่อง และแผงหลังเพื่อแก้อาการเครื่องดับจากความร้อน
  - 6.3.4 ซอฟต์แวร์ตรวจสอบประสิทธิภาพสุขภาพดิสก์และรอบเวลาการทำงานของมอเตอร์ด้วยตรรกะระบบ S.M.A.R.T.
  - 6.3.5 การบำรุงรักษาพรีเวนทีฟสำหรับฮาร์ดแวร์ต่อพ่วง (การล้างหัวฉีดหมึก Printer, การดูแลทำความสะอาดลูกกลิ้งสแกนเนอร์)
  - 6.3.6 การเปลี่ยนสารเหนี่ยวนำความร้อน (Thermal Paste Reapplication) สำหรับ CPU และวงรอบการทำความสะอาดพัดลมเคส
- 6.4 การเฝ้าระวังและวิเคราะห์ประสิทธิภาพระบบ (System & Network Monitoring)
  - 6.4.1 การตรวจเช็คทรัพยากรระบบผ่านแถบหน้าจอ Task Manager และ Resource Monitor เพื่อหาสาเหตุอาการเครื่องประมวลช้า
  - 6.4.2 แนะนำการติดตั้งเครื่องมือตรวจเช็คการไหลจราจรข้อมูลภายในระบบแลนเบื้องต้น (Wireshark และ Advanced IP Scanner)
  - 6.4.3 การวิเคราะห์ประวัติบันทึกสถานะของระบบและการแจ้งเตือนข้อบกพร่อง (Log Events) ผ่านการเรียกใช้ Windows Event Viewer

ใบงานปฏิบัติ 6: ดำเนินการบำรุงรักษาฮาร์ดแวร์ (เป่าฝุ่น/ทำความสะอาด) ใช้งานโปรแกรม Utility เพื่อเคลียร์พื้นที่ดิสก์ ตั้งค่าการสแกนไวรัส และสร้างจุด System Restore สำรองระบบ

## Unit 7: การวิเคราะห์ สรุปปัญหา และจัดทำรายงาน

- 7.1 กระบวนการวิเคราะห์และแก้ไขปัญหาอย่างเป็นระบบ (Troubleshooting Methodology)
  - 7.1.1 ศึกษาและประยุกต์ใช้ระเบียบแก้ไขปัญหา 6 สเตปตามมาตรฐานสากล: ระบุปัญหา (Identify), ตั้งสมมติฐาน (Theorize), ทดสอบหาสาเหตุ (Test), ลงมือวางแผนซ่อมแซม (Plan & Resolve), สอบทานระบบหลังทำ (Verify), และจัดเก็บเอกสารอย่างละเอียด (Document)
  - 7.1.2 เทคนิคทักษะการซักถามสอบประวัติจากผู้ใช้งานเครื่องเพื่อวินิจฉัยจุดที่ติดขัดเบื้องต้น (User Interview Techniques)
  - 7.1.3 การจำกัดวงและแยกแยะประเภทต้นกำเนิดความเสียหาย (Hardware defect, OS crash, Software bug, User mistake)
  - 7.1.4 การแปลสัญลักษณ์รหัสข้อผิดพลาดบนจอหน้าต่าง, สัญลักษณ์ Debug LED, เสียง Beep Code และหน้าต่างมรณะสีฟ้า (BSOD)
  - 7.1.5 การสืบหาสาเหตุโดยตรวจสอบการบันทึกประวัติการเปลี่ยนแปลงของระบบ (Configuration Changes, Windows Update)
- 7.2 การวิเคราะห์และแก้ไขปัญหาระบบเครือข่าย (Network Troubleshooting)
  - 7.2.1 การตรวจหาสาเหตุเชิงกายภาพ (Physical Layer): ตรวจสอบพอร์ตเชื่อมต่อ, การชำรุดของหัว RJ-45, ความเสียหายภายนอกของสาย LAN
  - 7.2.2 การกู้คืนสภาพปัญหาเครือข่ายเชิงซอฟต์แวร์ (Logical Layers): การแก้ไขไอพีชน (IP Conflict), ปัญหาหลุดการแจกไอพี (APIPA), DNS เสีย
  - 7.2.3 คำสั่งและโปรแกรมระดับนักพัฒนาระบบที่ใช้สืบเช็คและวิเคราะห์ไอพีและเซิร์ฟเวอร์ปลายทาง (nslookup, netstat, arp -a)
  - 7.2.4 การกู้ปัญหาและดูแลเครือข่าย Wi-Fi เบื้องต้น (ตรวจสอบช่องสัญญาณการชนกันของคลื่น, การแก้ไขรหัสผ่านสิทธิ์เชื่อมต่อผิด)
  - 7.2.5 การวิเคราะห์สิทธิ์การเข้าถึงข้อมูลแชร์ในเครือข่ายและการเชื่อมต่อโปรโตคอล (เช่น SMBv1 เทียบกับ SMBv2/v3)
- 7.3 การแก้ไขปัญหาอุปกรณ์ต่อพ่วงและการใช้แหล่งข้อมูลช่วยเหลือ
  - 7.3.1 วิธีแก้ไขปัญหาความขัดข้องระบบจัดพิมพ์ที่พบบ่อย (ล้างคิวงานพิมพ์ Print Spooler ค้าง, อัปเดตไดรเวอร์, จัดการแชร์พอร์ต)
  - 7.3.2 การถอดรหัสอ่านสัญลักษณ์ วิธีการเปิดอ่านและอ้างอิงคู่มือทางเทคนิคสำหรับการซ่อมแซมชิ้นส่วน (Service Manuals)
  - 7.3.3 วิธีค้นหาและใช้แหล่งเรียนรู้รวมองค์ความรู้จากผู้พัฒนาเว็บไซต์ และกลุ่มสังคมเว็บบอร์ดเทคนิคเพื่อแก้ปัญหายาก
  - 7.3.4 การประยุกต์ใช้ปัญญาประดิษฐ์ (AI Assistant) และเครื่องมือวินิจฉัยข้อบกพร่องระบบอัตโนมัติในการวิเคราะห์และแก้ไขปัญหา
- 7.4 การจัดการเอกสาร การบันทึกประวัติ และจัดทำรายงาน
  - 7.4.1 ความจำเป็นของการลงทะเบียนและการบันทึกประวัติทางเทคนิคลงคลังประวัติอุปกรณ์ (Asset Inventory & Maintenance Logs)
  - 7.4.2 โครงสร้างกลไกและขั้นตอนการเปิดรับส่งงานในระบบแจ้งซ่อมเทคนิคบริการ (IT Service Ticketing System Workflow)
  - 7.4.3 วิธีการลงลายมือเขียนรายงานผลการทดสอบการแก้ไขและการประเมินหาสาเหตุหลักรากเหง้าปัญหา (Root Cause Analysis - RCA)
  - 7.4.4 การวางข้อเสนอแนะ การจัดเก็บมาตรการแก้ไขปัญหาระยะยาว และการจัดทำคู่มือเบื้องต้นเชิงป้องกันให้กับองค์กร
  - 7.4.5 การสร้างคู่มือสั้นแนวทางการช่วยวินิจฉัยและซ่อมแซมตนเองเบื้องต้นสำหรับพนักงานองค์กร (User Self-Help Guide)

ใบงานปฏิบัติ 7: จับคู่ผู้เรียนเพื่อทำ Role-play (ช่างเทคนิค-ผู้ใช้งาน) โดยรับโจทย์สถานการณ์จำลองเครื่องเสียหรือเข้าเครือข่ายไม่ได้ ผู้เรียนต้องใช้กระบวนการวิเคราะห์เพื่อแก้ไขปัญหาจริง และเขียนรายงานแจ้งซ่อม/สรุปผลการแก้ไข (Maintenance Report)
